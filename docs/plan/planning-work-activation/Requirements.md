# Planning Work Activation Requirements

## REQ-001 — Planning activates Better Plan

Planning work enters Better Plan alongside coding and explicit implementation work. Other requests
continue to follow the user's instructions or receive a normal answer.

## REQ-002 — Activation surfaces stay consistent

The Hook guidance, Skill metadata, `SKILL.md`, native-main contract, and README expose the same
planning/coding/implementation activation boundary.

## REQ-003 — Progressive disclosure remains intact

Adding planning does not return lifecycle, acceptance, execution, regression, repair, audit,
completion, or next-Node policy to entry guidance.
