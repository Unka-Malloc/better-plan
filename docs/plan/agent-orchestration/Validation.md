## 子智能体门禁（新鲜实例）
1. **验收方案智能体（Acceptance Planner）**
   - 只能写：`docs/plan/agent-orchestration/Validation.md`（只读其他文档）。
   - 产出：本需求矩阵的“聚焦证明/最终证明/证据类型”与每条门禁说明。
   - 角色结束条件：提交可追踪的文档更新后退出，不执行代码。
2. **执行智能体（Executor）**
   - 仅按主智能体提供的 Node ID 有序列表执行：如 `["06f25b2d...", "95aa9e02...", "5470aaad..."]`（示例格式）。
   - 顺序规则：严格按列表顺序；不得并行；不得替换未给定 Node。
   - 角色结束条件：每一 Node 的结果与回归证据完整回传给主智能体。
3. **停止审计智能体（Stop Auditor）**
   - 只读审计：不得修改任何实现文件、不得提交修复。
   - 核验：按 `Validation.md` 的最终证明逐项对照，不合格需返回可复现发现点与建议修复边界。
   - 角色结束条件：返回 PASS 或发现问题清单与优先级。

## 异常与重试规则
- **超时阈值**：子智能体未在 12 分钟内返回结果视为超时失败；主智能体可重新分派同一角色新实例一次并附带失效上下文摘要。
- **失败阈值**：失败 2 次后，主智能体暂停推进并要求用户确认；若问题可自愈可重开执行轮次。
- **发现问题后的流程**：执行发现不达标→主智能体更新 `Execution Gate` 记录→主执行轮次修复→再调度执行智能体（有序 Node）→再触发停止审计智能体进行二次只读审计。
- **全量回归限制**：仅允许在最后一次问题闭环后、所有角色门禁通过后，执行一次 `full` 回归；之后不得再次重复全量回归。

## 里程碑完成判断
- **阶段验收**：REQ-001～REQ-006 以文档与流程闭环通过为准。
- **交付验收**：REQ-007～REQ-009 以停止审计智能体 PASS、受控隐私边界、状态一致性和唯一全量回归为准。
